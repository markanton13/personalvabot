# Security Policy

## Reporting a vulnerability

Do not open a public issue for:

- exposed tokens or credentials;
- private client or worker data;
- database or backup exposure;
- installer tampering;
- authentication or permission bypass;
- destructive data-loss behavior.

Report sensitive issues privately to the project owner through the contact information on:

https://markbadong.com

## Never share publicly

- Discord bot tokens;
- application credentials;
- `.env` files;
- SQLite databases;
- backups;
- logs containing private data;
- client records;
- invoices or attendance records.

## Supported version

Security review currently focuses on the latest Windows Desktop Beta release.
